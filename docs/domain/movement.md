# Movement

> 対応コード: `server/src/systems/movement.ts`, `server/src/utils/collision.ts`, `client/src/input/mouse.ts`, `client/src/input/keyboard.ts`
> コードと本ドキュメントに乖離がある場合はコードを優先すること。

## クリック移動フロー

1. プレイヤーが地面をクリック → `move { target }` をサーバへ送信
2. サーバが移動距離を `MAX_MOVE_DIST（300px）` でクランプ
3. `moveQueue`（最大5件）に追加
4. **ピボットターン**: 目標方向と現在の車体向きに角度差がある場合、まず旋回
5. 旋回完了後に前進開始
6. 目標地点に到着（距離 ≤ `MOVE_SPEED`）したらクールダウン発動

---

## ピボットターン

- `hullAngle` を目標方向に向けて毎 tick `HULL_ROTATION_SPEED（π/15 rad）` ずつ回転
- 砲塔（`turretAngle`）も同方向に追従（速度: π/10 rad/tick）
- 角度差が 0.05rad 以下になったら前進フェーズへ移行
- 旋回中: `isRotating === true`、前進中: `isMoving === true`

---

## 移動予約

- 移動中・クールダウン中でも新たなクリックを `moveQueue` に積める（上限5件）
- `Z` キー（`moveCancelOne`）で末尾の予約を1件削除
- 壁またはタンクに衝突した場合: 移動中断 → 現在のターゲットを破棄 → クールダウン発動

---

## プレイヤー間衝突

- 移動先が他のタンク（半径 18px × 2.2 ≒ 39.6px 以内）と重なる場合は原則として移動を中断する
- **例外（スタック回避ロジック）**:
  - `respawnCooldownUntil`（リスポーン直後の1.5秒間）中は、衝突判定をスキップしてすり抜けが可能
  - 互いに重なっている状態でも、「現在よりも離れる方向（`nextDistSq >= currentDistSq`）」への移動であれば許可される（密集からの脱出を保証）
- 衝突処理: 移動中断 + クールダウン発動

## カメラ操作（クライアント）

| 操作 | 動作 |
|---|---|
| `↑` / `↓` / `←` / `→`（Arrow キー） | カメラパン（カメラの回転方向に合わせた移動） |
| マウスホイール | ズームイン/アウト（0.3〜3.0倍） |
| `+` / `=` キー | ズームイン |
| `-` キー | ズームアウト |
| `Q` キー | カメラ左回転 |
| `E` キー | カメラ右回転 |
| `Space` キー | 自機位置にスナップ（ズーム1.0, 回転0） |
| ミニマップクリック | クリック地点へ移動指示を送信 |

### カメラ仕様

- 自機の自動追従なし（TankMatch 仕様準拠）
- カメラは常にワールド座標 `(camera.x, camera.y)` を中心に描画
- 入室時にカメラが自機のスポーン位置に自動移動
- チャット入力中は Arrow キー / Q / E / Space / マウスクリックがすべて無効化される

---

## 制約

- 移動距離の最大値はサーバがクランプする（クライアントの値を信用しない）
- クールダウン中の移動開始はサーバが拒否する（移動予約は受け付ける）
- 壁内部への移動はサーバが中断させ、クールダウンを発動する
