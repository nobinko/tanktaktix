## Phase 4: アイテムシステム刷新（完了）

アイテムシステムの根本的な見直しと新アイテム・新アクションの追加。

- **固定プール方式**: マップごとにアイテム総量を12個に固定（medic/ammo/heart/bomb/rope/boots 各2個）
- **アイテム破壊**: 弾丸命中でアイテムを破壊可能（別地点に再スポーン）
- **所持上限チェック**: HP 満タンなら回復系取得不可、ammo 上限40発、rope 上限2本、bomb/boots 上限1個
- **新アイテム4種**: heart（HP全回復）、bomb（爆風3倍の特殊弾）、rope（AIM方向にアイテム/フラッグ回収）、boots（移動速度1.5倍・3回移動分）
- **フラッグ挙動改善**: 被弾ドロップ → その場に落下、立ち止まりでは再取得しない仕様
- **ダメージ表現**: HP 段階（5段階）で車体の見た目が変化（非対称な多角形欠損、装甲の剥がれ、HP40%以下での火災エフェクト）。HPバーは廃止。
- **AIM アクション派生4種**: R（ロープ射出）、A（アモ投げ）、H（ヘルス投げ）、F（フラッグ投げ）
- **UI 改善**: 全画面化（100vw x 100vh）、ブラウザネイティブポップアップ廃止 → カスタムモーダル

## Phase 5: 新機能・品質改善（進行中）

> スケール方針・最適化の設計メモは [architecture.md](./architecture.md) を参照。

### 完了済み

#### 5-1. アイテム当たり判定の抜本改修
アイテム接触時の移動クランプ、クールダウン、所持上限チェックを含む当たり判定ロジックの根本的見直しを実施。

#### 5-2. サウンドエフェクト（SE）
BGM なし（配信・実況フレンドリー設計）。`SoundManager` ベースで射撃・爆発・フラッグ・アイテム取得・UI 操作音を実装。残弾0でもボム所持時はボム専用弾として発射可能に改善。Setting モーダルのミュート連動。

#### 5-3. マルチロビー
ロビーを Main Lobby / Sub Lobby 1 / Sub Lobby 2 に分割。ルーム・チャット・プレイヤーリストをロビーごとに完全隔離。全遷移（参加/退出/切断/スペクター）でリスト即時同期。

#### 5-6. ゲームプレイルール調整 / UI 整理
弾薬表示を自機 HUD のみに変更、ロビー残り時間の実時間併記、ランダム名を `GP-1234` / `bot-1234` 形式に統一、ロープ使用時のシステムコメント非表示、旗を弾丸の障害物化、カメラ移動制限のマージン追加。

#### 5-8. カスタムマップ制作
旧テストマップ（alpha〜epsilon, test S/M/L）を全削除し、ゲームプレイ重視の2マップに刷新:
- **Riverside** (1600×1200): 中央を縦断する川と2本の橋。橋を挟んだ攻防戦。斜めワンウェイ付き。
- **Fortress** (1800×1200): 4隅の水場、中央アイランド、各チーム2拠点型のベース構造。

マップ設計の基本原則（点対称/線対称、旗複数配置、配置センス）を `docs/inbox.md` に記録。

#### 5-14. 複数フラッグ CTF 対応
`Flag` 型に `baseX`/`baseY` を追加し、各フラッグが固有の元位置を保持。`room.ts` のフラッグ初期化を `map()` ベースに変更。`ctf.ts`・`projectiles.ts` のリセット処理を `f.baseX/f.baseY` に統一。1チームに複数フラッグを配置でき、戦略的な旗分散攻撃が可能に。

#### 5-9. ロビー機能追加（マップサムネイル）
ロビーの部屋一覧に、各ゲームモード・マップ構成のサムネイル（壁・スポーン位置）を表示する機能を追加。リバー・ブリッジのサムネイル描画対応。

#### 5-10. タイトル画面リデザイン
diep.io 方式を採用。Canvas（ゲームエンジン）をタイトル画面の背景として使用（プレイヤー在籍中は観戦モード、不在時はマップのみ表示）。ログイン UI を半透明オーバーレイパネルに変更。タイトルロゴのグラフィック強化。

#### 5-11. ロビー画面リデザイン
TankMatch 参考の構造 x モダンカード UI。左メインに部屋カード、右上チャット、右下プレイヤー一覧、ボトムバーにアクション配置。ロビーヘッダーに賑わいインジケーター追加。Create Room をモーダル方式に変更。

#### 5-12. ゲームルームオプション4種
Create Room モーダルに4つの特殊ルールオプションを実装: Team Select（チーム手動選択）、Instant Kill（即死モード）、No Item Respawn（アイテム再出現なし）、No Shooting（射撃禁止）。

#### 5-15. サーバ・クライアント通信最適化＆描画補間 (大規模人数対応)
大規模人数対応に向けた基盤改修。クライアント側でのエンティティ座標補間（Interpolation）を導入して描画速度と滑らかさを向上させ、サーバーからは初回のみマップ全体データを送信し以降は差分データと位置情報のみを送るプロトコルに変更（ペイロード圧縮とキャッシュ化）。さらにサーバー側Tickでマンハッタン距離を用いた衝突判定の足切り（Early Reject）を導入しCPU使用率を最適化。

#### 5-16. 大規模人数向けのリスポーン最適化とスタック回避
30vs30などでリスポーン時に多数のプレイヤーが重なってスタックする問題を解決。スポーン地点に `radius`（半径）の概念を導入してエリア内でランダムに散開させるように拡張。さらに、リスポーン直後の1.5秒間に無敵・すり抜け・移動許可を組み合わせ、互いの衝突判定（押し出しあう挙動）を回避することで、万が一重なっても自然と密集から抜け出せる物理ロジックを実装。

#### 5-17. スモークアイテム実装
新アイテム「smoke」を追加。取得上限1個で、クールダウン中にも使用でき、使用によるクールダウンも発生しない。
足元への直接展開（Sキー）と、エイム方向への投擲（エイム+Sキー）に両対応。
展開された煙幕は20秒間持続し、範囲内（半径130）のタンクをゲームシステム的（isHidden: true）にも視覚的にも完全に隠蔽する。
フィールド上のスモークアイテムに弾丸や爆風が命中すると、そのアイテムが直ちに起爆し煙幕を展開するギミックも搭載。

#### 5-5. マップエディタ
ロビーの「MAP EDITOR」ボタンから開けるフルスペックのビジュアルマップエディタを実装。
- **マップサイズ選択**: Small (800×600), Medium (1200×900), Large (1800×1200), Custom（任意サイズ）
- **パレット**: WALLS（wall/bush/water/house/oneway/river/bridge）、SPAWN（red/blue）、FLAGS（red/blue）、ITEMS（全6種）、PREFABS（house-s/m/l・bases・rivers・bridges）
- **対称配置**: NONE / H（左右）/ V（上下）/ PT（点対称）
- **操作**: ドラッグ描画、移動（選択後ドラッグ）、リサイズ（8ハンドル）、回転（R/Q キー ±15°）、削除（Del）
- **アンドゥ/リドゥ**: Ctrl+Z / Ctrl+Y（50ステップ）
- **グリッドスナップ**: Grid Snap チェックボックスで切替（グリッドサイズ 20px）
- **ズーム・パン**: スクロールでズーム、Space+ドラッグでパン
- **JSON インポート/エクスポート**: LOAD JSON で既存 MapData を読み込み、EXPORT JSON でクリップボードに書き出し
- **PLAY TEST**: エディタ上のマップをカスタムマップとして即座に部屋作成してプレイ可能
- **リバーエルボー描画**: rv-elbow-* 系プリファブは Canvas arc API でドーナツセクター形状（スムーズ曲線）として描画
