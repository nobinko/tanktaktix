# ゲームメカニクス仕様書

このドキュメントは TankTaktix のゲームメカニクスを定義する。
数値は `server/src/index.ts` の定数と一致していること。乖離が生じた場合はコードを優先する。

---

## 基本パラメータ

| パラメータ | 値 | 備考 |
|---|---|---|
| ワールドサイズ | 1800 × 1040 px | 4マップ共通 |
| Canvas サイズ | 1200 × 675 px（16:9） | ワールドの約 2/3 を表示 |
| マップ | alpha / beta / gamma / delta / epsilon | `room.mapData` で動的取得 |
| サーバtick | 50ms (20Hz) | `TICK_MS` |
| 移動速度 | 6 px/tick | `MOVE_SPEED` |
| タンクサイズ | 半径18px（衝突円）/ 26×20px（回転矩形） | `TANK_SIZE` |
| 移動キュー上限 | 5件 | `MOVE_QUEUE_MAX` |
| 最大移動距離（1回） | 300px | `MAX_MOVE_DIST` |

### マップ一覧

| マップID | レイアウト | 壁数 | 特徴 |
|---|---|---|---|
| **alpha** | クラシック | 5 | 左右対称の縦壁 + 中央アイランド |
| **beta** | アーバン | 8 | 6本のピラーで3列コリドー + 横カバー |
| **gamma** | フォート | 8 | 内側に正方形の要塞 + 外側カバー |
| **delta** | 自然 | 6（壁2 + ブッシュ2 + 水場2） | ブッシュと水場が点対称に配置（CTF向け） |
| **epsilon** | インポシブル | 約18 | ハウス、コの字型ベース、ワンウェイ壁が配置された競技用難関マップ |

## ゲームモード

| モード | 選択値 | 勝利条件 |
|---|---|---|
| **デスマッチ** | `deathmatch` | 制限時間終了時にチームスコアが高い方が勝利 |
| **CTF (旗取り)** | `ctf` | 制限時間終了時に旗奪取スコアが高い方が勝利 |

ルーム作成時に `gameMode` を選択する。デフォルトは `ctf`。

## クールダウン（硬直）仕様

すべてのクールダウンは「カウント」として戦車の頭上に表示される。
- **1カウント = 300ms**
- カウント中はすべての行動（移動・射撃）が不能（移動予約のみ受け付ける）

### アクション別クールダウン

| アクション | 条件 | クールダウン | カウント | 定数 |
|---|---|---|---|---|
| **ショートムーブ** | 移動距離 < 200px | 1500ms | 5 | `COOLDOWN_SHORT_MS` |
| **ロングムーブ** | 移動距離 ≥ 200px | 2100ms | 7 | `COOLDOWN_LONG_MS` |
| **射撃** | 常に固定 | 1800ms | 6 | `ACTION_COOLDOWN_MS` |

- **距離判定**: 移動元（最後の予約地点 or 現在地）から移動先までの直線距離
- クールダウンは到着時に発動する（移動中は別途 `isMoving` で管理）

### UI表示

- **移動中**: カウント非表示（`isMoving === true` の間）
- **到着後 / 射撃後**: 即座にカウントダウン開始
- **カウントダウン**: 300ms ごとに 1 減少（例: 7 → 6 → 5 ... → 0）
- **0 になった瞬間**: 次のアクションが可能になる

## 移動メカニクス

### クリック移動フロー

1. プレイヤーが地面をクリック → `move { target }` を送信
2. サーバが移動距離を `MAX_MOVE_DIST(300px)` でクランプ
3. `moveQueue`（最大5件）に追加
4. **ピボットターン**: 目標方向と現在の車体向きに角度差がある場合、まず旋回する
5. 旋回完了後に前進開始
6. 目標地点に到着（距離 ≤ `MOVE_SPEED`）したらクールダウン発動

### ピボットターン

- `hullAngle` を目標方向に向けて毎tick `HULL_ROTATION_SPEED(π/15 rad)` ずつ回転
- 砲塔（`turretAngle`）も同方向に追従（速度: `π/10 rad/tick`）
- 角度差が `0.05rad` 以下になったら前進フェーズへ移行
- ピボットターン中は `isRotating === true`、前進中は `isMoving === true`

### 移動予約

- 移動中・クールダウン中でも新たなクリックを `moveQueue` に積める
- `Z` キー（`moveCancelOne`）で末尾の予約を1件削除
- 壁またはタンクに衝突した場合: 移動中断 → 現在のターゲットを破棄 → クールダウン発動

### プレイヤー間衝突

- 移動先が他のタンク（半径18px × 2.2 ≒ 39.6px 以内）と重なる場合は移動を中断
- 衝突処理: 移動中断 + クールダウン発動

## 射撃メカニクス

### AIM（スリングショット方式）

1. 自分のタンク上でマウスボタン押下 → AIMモード開始
2. ドラッグ中: ドラッグ方向と**逆方向**に射撃ガイドライン表示
3. 自機の半径（18px）以内に戻してボタンを離す → **キャンセル**
4. 半径外でボタンを離す → **発射**
   - 発射方向 = ドラッグベクトルの逆方向（単位ベクトル）

### 射撃条件チェック（サーバ）

- クールダウン中 → 拒否
- 移動中 (`isMoving === true`) → 拒否
- リスポーン無敵期間中 → 拒否
- 弾薬が0 → 拒否

### 弾丸パラメータ

| パラメータ | 値 | 定数名 |
|---|---|---|
| 弾速 | 220 px/s | `BULLET_SPEED` |
| 射程 | 無限（99999px） | `BULLET_RANGE` |
| 弾丸半径 | 4px（ボム弾: 6px） | `BULLET_RADIUS` |
| 爆発半径 | 40px（ボム: 120px） | `EXPLOSION_RADIUS` |
| 1発ダメージ | 20 | `EXPLOSION_DAMAGE` |

### 命中判定

- 弾丸の前フレーム位置 → 今フレーム位置のレイと、タンクの**回転矩形**（26×20px）の交差判定
- アルゴリズム: Liang-Barsky（弾丸をタンクのローカル座標系に変換して AABB チェック）
- 壁衝突・マップ外への到達で爆発する

## HP / 弾薬

| パラメータ | 初期値 | 変化 |
|---|---|---|
| HP | 100 | 被弾ごとに 20 減少。0 で即死 |
| 弾薬 | 20発 | 射撃ごとに -1。0 で射撃不可。最大40（アイテムで回復） |

アイテムによる回復については「アイテム」セクションを参照。

## リスポーン

### インスタントリスポーン（現行仕様）

- HP が 0 になると**即座に**リスポーン地点に移動
- リスポーン後 **1500ms** の無敵期間（`respawnCooldownUntil`）
- 無敵期間中: 被弾ダメージなし、弾丸当たりなし、行動不可
- 無敵期間中はクライアント描画でタンクが半透明（α=0.5）
- スコア・キル数・死亡数はリスポーン後も維持される（初回参加時のみリセット）

### リスポーン地点

- チームのスポーンポイントを基準に **±40px のランダム範囲** 内でスポーン
- 壁・他プレイヤーとの衝突を回避するため最大20回再試行
- 全試行失敗時はスポーンポイントの初期座標にフォールバック

## スコアリング

### デスマッチ

| イベント | 変化 |
|---|---|
| 敵を撃破 | 撃破者 +1点 / 撃破者のチームスコア +1 |
| 自爆 | 変化なし |
| 死亡 | 変化なし（ペナルティなし） |
| 時間切れ | チームスコアが高い方が勝利（同点は Draw） |

### CTF（旗取り）

| イベント | 変化 |
|---|---|
| 敵の旗を取得 | フラッグキャリア状態になる |
| 自陣スポーンゾーン内で**完全停止** | チームスコア +5、個人スコア +5、旗リセット |
| キャリアが被弾（致死でなくても） | 旗をドロップ → ベース位置に即時リセット |
| キャリアが死亡 | 旗をドロップ → ベース位置に即時リセット |
| 時間切れ | チームスコアが高い方が勝利（同点は Draw） |

**「完全停止」ルール**: 旗を持って自陣に戻っただけではスコアにならない。
自陣の**スポーンゾーン（スポーン地点を中心とした 200×200 の矩形エリア）**内で `isMoving === false && isRotating === false` になった瞬間にスコアが加算される。

## フレンドリーファイア

- **自爆**: ダメージあり
- **味方への攻撃**: ダメージなし（弾丸は貫通扱い）
- **敵への攻撃**: 通常どおりダメージあり

## アイテム（B-1 / Phase 4）

### アイテムタイプ

| タイプ | 効果 | 所持上限 | 取得不可条件 |
|---|---|---|---|
| **medic**（回復キット） | HP +20（最大100） | ― | HP = 100 |
| **ammo**（弾薬ボックス） | 弾薬 +10（最大40） | ― | ammo ≥ 40 |
| **heart**（ハート） | HP 全回復（= 100） | ― | HP = 100 |
| **bomb**（ボム） | 次の射撃が3倍爆発半径（内側60/中間40/外側20ダメージ） | 1個 | 既に所持中 |
| **rope**（ロープ） | 射出して遠距離のアイテムや旗を回収（1本=200px / 2本=300px） | 2本 | 既に2本所持 |
| **boots**（ブーツ） | 移動速度 1.5倍（3回の移動到着で消費） | 3チャージ | ― |

### スポーンルール（Phase 4: 固定プール方式）

- 各マップ開始時に **12個**（各タイプ2個ずつ）をランダム位置にスポーン
- 取得またはアイテム破壊後、同タイプ1個がランダム位置にリスポーン
- 壁・水場の内部にはスポーンしない
- **取得判定**: タンク中心からアイテム中心まで `TANK_SIZE + ITEM_RADIUS (33px)` 以内で自動取得

### アイテムと弾丸の相互作用（Phase 4-2）

- 弾丸がアイテムに命中するとアイテムが破壊される（リスポーン対象）
- ロープ弾・パス弾はアイテムを破壊しない

### 描画

- medic: 緑色の正方形 + 白い十字
- ammo: 黄/茶の正方形 + 弾丸アイコン
- heart / bomb / rope / boots: 各タイプ別のスプライト描画
- ミニマップにもドット表示

## 地形タイプ（B-2）

| タイプ | 通行 | 弾丸 | 特殊効果 |
|---|---|---|---|
| **wall** | 不可 | 遮断（爆発） | なし |
| **bush** | 可 | 貫通 | 内部のプレイヤーを隠密状態にする |
| **water** | 不可 | 貫通 | なし |
| **house** | 不可 | 遮断（爆発） | 建物型の外観を持つ障害物 |
| **oneway**| 不可 | 条件付き透過 | 指定された方向へ撃った弾だけが透過し、逆方向の弾は防ぐ一方通行の壁 |

### 隠密（B-5）

- bush 内にいるプレイヤーは `isHidden = true` となる
- 隠密中は**敵チームに座標が送信されない**（ミニマップにも映らない）
- **bush 内で射撃しても隠密は解除されない**（bush 外に出た瞬間に `isHidden = false`）
- 味方チームには常に可視

## CTF モード詳細（B-4）

### フラッグルール

- 各チームの陣地にフラッグ（旗）が1本ずつ配置される
- 敵チームの旗の位置に接触すると**旗を取得**（キャリア状態）
- キャリアは頭上に 🚩 アイコンが表示される
- **自陣のスポーンゾーン内**で完全停止するとチームスコア +5、旗がリセットされる
- キャリアが**ダメージを受ける（致死でなくても）か死亡**すると、旗はベース位置（`flagPositions` の初期位置）に即時リセットされる
- ロープでキャリアから旗を奪取可能（味方からも奪取可能）
- AIM+F で旗をパス弾として投げ、味方がキャッチ可能

### ミニマップ表示

- 旗はミニマップ上に赤/青の円として表示される

---

## 再接続（B-3）

- ブラウザリロード / 切断から **60秒以内** に再接続すると同セッションに復帰
- クライアントは `localStorage` に `tt_id` を保存
- `login` メッセージに `id` フィールドを付与して前セッションの ID を送信
- サーバ側で `disconnectedPlayers` マップに 60秒間セッションを保持

## カメラ操作（クライアント）

| 操作 | 動作 |
|---|---|
| `↑` / `↓` / `←` / `→`（Arrow キー） | カメラパン（カメラの回転方向に合わせた移動） |
| マウスホイール | ズームイン/アウト（0.3〜3.0倍） |
| `+` / `=` キー | ズームイン |
| `-` キー | ズームアウト |
| `Q` キー | カメラ左回転 |
| `E` キー | カメラ右回転 |
| `Space` キー | 自機位置にスナップ（ズーム1.0, 回転0） |
| ミニマップクリック | クリック地点へ移動指示を送信 |

- **自機の自動追従なし**（TankMatch 仕様準拠）
- カメラは常にワールド座標 `(camera.x, camera.y)` を中心に描画
- **入室時にカメラが自機のスポーン位置に自動移動**
- チャット入力中は Arrow キー / Q / E / Space / マウスクリックがすべて無効化される

## AIMアクション派生（Phase 4-7）

AIMモード中に特定キーを押すと、射撃の代わりに特殊行動を実行する。
すべてのAIMアクションは射撃と同じクールダウン（1800ms / 6カウント）を共有する。

| キー | アクション | 消費 | 効果 |
|---|---|---|---|
| **R** | ロープ射出 | rope 1本 | 射出方向にロープ弾を飛ばし、着弾地点付近のアイテム/旗を自機に引き寄せる |
| **A** | 弾薬パス | ammo 5発 | 射出方向に弾薬パス弾を飛ばし、味方が接触すると ammo +5 |
| **H** | 回復パス | HP 20 | 射出方向に回復パス弾を飛ばし、味方が接触すると HP +20 |
| **F** | 旗パス | 所持中の旗 | 旗をパス弾として射出。味方がキャッチ可能 |
