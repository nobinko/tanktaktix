# 受入条件 (Acceptance Criteria)

本ドキュメントは TankTaktix の変更を受け入れる際の判定基準を定義する。
ランクは **A（必須）/ B（推奨）/ C（将来）** の3段階で、deep-research-report の MVP 区分（Must / Should / Later）に対応する。

> **運用ルール**: PR は A ランクを全て満たす場合のみマージ可。B は「満たさなくても OK だが、壊してはならない」。C は参考情報。

---

## A ランク — MVP 必須（Must）

すべての PR で以下を満たすこと。既存の A 項目を壊す変更は NG。

### A-1. ゲストログイン
- [x] GuestPlay → Name 入力 → LogIn で接続が成功する
- [x] 未入力時はランダム4桁名が割り当てられる
- [x] ログイン中に "Synchronizing …ms" が表示される

### A-2. ロビー
- [x] ゲーム一覧が表示される
- [x] New game / Help / Setting / Exit ボタンが存在する
- [x] ロビーチャットが送受信できる

### A-3. ルーム作成・参加
- [x] GameID / PlayerMax / Time / Password を指定してルーム作成できる
- [x] 一覧からルーム選択 → Join で参加できる
- [x] パスワード付きルームはパスワード無しで参加できない

### A-4. 対戦コア — 移動
- [x] 左クリックで移動指示（十字マーク → 移動開始）
- [x] 最大移動距離制限が機能する
- [x] 移動中/カウント中のクリックで移動予約される
- [x] Z キーで予約を1つキャンセルできる

### A-5. 対戦コア — 射撃
- [x] 自機上で左ボタン押し → ドラッグ → 離すで弾が発射される
- [x] 弾は飛翔し、壁/敵への命中で爆発する（即爆発しない）
- [x] 残弾 0 で射撃不可になる

### A-6. 行動カウント
- [x] 行動後にカウント(5→0)が表示され、カウント中は行動不可
- [ ] スモーク展開は行動カウントが発生しない（将来実装時）

### A-7. HP / 弾薬
- [x] 初期 HP 100%、被弾ごとに 20% 減少
- [x] HP 0 で死亡 → リスポーン（HP/弾薬リセット）
- [x] 初期弾薬 20、射撃ごとに -1

### A-8. チーム戦スコア
- [x] 敵撃破で撃破者に個人 +1点、撃破者のチームスコア +1
- [x] 自爆・死亡はスコア変化なし（ペナルティなし）
- [x] スコア・キル・デスはリスポーン後も維持（参加時のみリセット）
- [x] 制限時間終了でチームスコアが高い方が勝利（同点は Draw）
- [x] フラッグ持ち帰りで +5点（チームスコア）
- [ ] 個人戦モードのスコア（命中+1 / 撃破+5 / 死亡-5）は将来実装

### A-9. チーム戦（基本）
- [x] Red / Blue に分かれる
- [x] 味方への攻撃はダメージ無し（FF 無効）

### A-10. 視点
- [x] 自機移動でカメラが自動追従しない
- [x] Arrow キーでカメラ移動、Q/E で回転、Space でリセット

### A-11. チャット
- [x] T キーでチャットを開く
- [x] チャットUI表示中は他操作が一切不能
- [x] Enter で送信、チャットログが左下に表示

### A-12. サーバ権威
- [x] 移動・射撃・HP・スコアはサーバが計算する
- [x] クライアントからの不正入力（硬直中の行動等）はサーバが拒否する

## B ランク — MVP 推奨（Should）

壊してはならないが、未実装でも PR マージ可。

### B-1. アイテム
- [x] メディカルキット / アモパッケージの実装
- [x] フィールド上への自動スポーンと衝突判定

### B-2. 障害物
- [x] ブッシュ（隠蔽機能：中にいる間は他チームから見えない）
- [x] 水場（通行不可だが、弾丸は通過可能）

### B-3. 再接続・復帰
- [x] WS 切断後、60秒以内であればブラウザ更新等から元のセッションに自動復帰

### B-4. チーム戦（フラッグ）
- [x] フラッグ取得 → 自陣スポーンゾーン（200×200）内での **完全停止** でチームスコア +5
- [x] フラッグキャリアがダメージを受けるか死亡すると旗はベース位置（`flagPositions`）に即時リセット

### B-5. ステルス可視性
- [x] ブッシュ内の自機は敵チームから不可視
- [x] 不可視対象の座標を敵クライアントへ送信しない（サーバ側でのフィルタリング）
- [x] ブッシュ内で射撃しても隠密は解除されない（Phase 3.5 で実装済み）

### B-6. 観戦モード
- [x] ルームに Watch で参加し、試合を観戦できる

### B-7. AIM 派生アクション（Phase 4-7）
- [x] AIM+R（ロープ射出）、AIM+A（アモ投げ）、AIM+H（ヘルス投げ）、AIM+F（フラッグ投げ）を実装

## 判定ルール

| ランク | PR マージ条件 |
|---|---|
| **A** | 全項目を満たす（既存 A 項目を壊さない） |
| **B** | 壊さない（未実装 OK） |
| **C** | 判定対象外 |

### 確認方法
1. **自動テスト**: 将来 `npm test` でカバー（現在未整備）
2. **手動確認**: devサーバで該当機能を目視操作
3. **コードレビュー**: サーバ権威の検証ロジックが含まれているか
