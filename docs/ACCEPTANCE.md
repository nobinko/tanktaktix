# 受入条件 (Acceptance Criteria)

本ドキュメントは TankTaktix の変更を受け入れる際の判定基準を定義する。
ランクは **A（必須）/ B（推奨）/ C（将来）** の3段階で、deep-research-report の MVP 区分（Must / Should / Later）に対応する。

> **運用ルール**: PR は A ランクを全て満たす場合のみマージ可。B は「満たさなくても OK だが、壊してはならない」。C は参考情報。

---

## A ランク — MVP 必須（Must）

すべての PR で以下を満たすこと。既存の A 項目を壊す変更は NG。

### A-1. ゲストログイン
- [ ] GuestPlay → Name 入力 → LogIn で接続が成功する
- [ ] 未入力時はランダム4桁名が割り当てられる
- [ ] ログイン中に "Synchronizing …ms" が表示される

### A-2. ロビー
- [ ] ゲーム一覧が表示される
- [ ] New game / Help / Setting / Exit ボタンが存在する
- [ ] ロビーチャットが送受信できる

### A-3. ルーム作成・参加
- [ ] GameID / PlayerMax / Time / Password を指定してルーム作成できる
- [ ] 一覧からルーム選択 → Join で参加できる
- [ ] パスワード付きルームはパスワード無しで参加できない

### A-4. 対戦コア — 移動
- [ ] 左クリックで移動指示（十字マーク → 移動開始）
- [ ] 最大移動距離制限が機能する
- [ ] 移動中/カウント中のクリックで移動予約される
- [ ] Z キーで予約を1つキャンセルできる

### A-5. 対戦コア — 射撃
- [ ] 自機上で左ボタン押し → ドラッグ → 離すで弾が発射される
- [ ] 弾は飛翔し、壁/敵への命中で爆発する（即爆発しない）
- [ ] 残弾 0 で射撃不可になる

### A-6. 行動カウント
- [ ] 行動後にカウント(5→0)が表示され、カウント中は行動不可
- [ ] スモーク展開は行動カウントが発生しない（将来実装時）

### A-7. HP / 弾薬
- [ ] 初期 HP 100%、被弾ごとに 20% 減少
- [ ] HP 0 で死亡 → リスポーン（HP/弾薬リセット）
- [ ] 初期弾薬 20、射撃ごとに -1

### A-8. チーム戦スコア
- [ ] 敵撃破で撃破者に個人 +1点、撃破者のチームスコア +1
- [ ] 自爆・死亡はスコア変化なし（ペナルティなし）
- [ ] スコア・キル・デスはリスポーン後も維持（参加時のみリセット）
- [ ] 制限時間終了でチームスコアが高い方が勝利（同点は Draw）
- [ ] フラッグ持ち帰りで +5点（フラッグ戦実装時に追加）
- [ ] 個人戦モードのスコア（命中+1 / 撃破+5 / 死亡-5）は将来実装

### A-9. チーム戦（基本）
- [ ] Red / Blue に分かれる
- [ ] 味方への攻撃はダメージ無し（FF 無効）

### A-10. 視点
- [ ] 自機移動でカメラが自動追従しない
- [ ] WASD でカメラ移動、Q/E で回転、Space でリセット

### A-11. チャット
- [ ] T キーでチャットを開く
- [ ] チャットUI表示中は他操作が一切不能
- [ ] Enter で送信、チャットログが左下に表示

### A-12. サーバ権威
- [ ] 移動・射撃・HP・スコアはサーバが計算する
- [ ] クライアントからの不正入力（硬直中の行動等）はサーバが拒否する

## B ランク — MVP 推奨（Should）

壊してはならないが、未実装でも PR マージ可。

### B-1. アイテム
- [ ] メディカルキット / アモ / ボム / ロープ / スモーク / 地雷 / フラッグの実装
- [ ] 所持上限の制約

### B-2. 障害物
- [ ] ワンウェイ / ブッシュ / ハウス / ベース / ブリッジ / リバーの通過可否

### B-3. 再接続・復帰
- [ ] WS 切断 → 自動再接続 → rejoin → スナップショット復帰

### B-4. チーム戦（フラッグ）
- [ ] フラッグ取得 → 自軍基地帰還で +5
- [ ] フラッグ保持中の被弾でフラッグリセット

### B-5. ブッシュ / スモーク可視性
- [ ] ブッシュ内の自機は他プレイヤーから不可視
- [ ] 不可視対象の座標を他クライアントへ送信しない

### B-6. 観戦モード
- [ ] ルームに Watch で参加し、試合を観戦できる

## 判定ルール

| ランク | PR マージ条件 |
|---|---|
| **A** | 全項目を満たす（既存 A 項目を壊さない） |
| **B** | 壊さない（未実装 OK） |
| **C** | 判定対象外 |

### 確認方法
1. **自動テスト**: 将来 `npm test` でカバー（現在未整備）
2. **手動確認**: devサーバで該当機能を目視操作
3. **コードレビュー**: サーバ権威の検証ロジックが含まれているか
