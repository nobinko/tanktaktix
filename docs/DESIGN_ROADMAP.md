## 1. グランドデザイン方針

### コンセプト
- **視認性最優先**：TankMatch 参考の明るい統一パレット（Phase 2 完了）
- **タクティカル＋クラシック**：軍事的な重厚感 × アナログな温かみ
- **配信・実況フレンドリー**：画面が見やすく、音がうるさくない設計

### 3画面構成
```
タイトル画面  ──→  ロビー画面  ──→  ゲーム画面
  世界観の入口       マッチメイキング    コアゲームプレイ
```

サブ画面：モーダル / リザルト / ヘルプ / 設定

## 2. 画面別ビジュアル設計

### 🏰 タイトル画面（現: ログイン画面）

**現状**
- CSSパネル + テキストフォーム（質素）
- 背景はただのクリーム色

**目指す姿：diep.io方式（裏でゲームが動く）**

```
┌─────────────────────────────────────┐
│  [実際のゲーム画面がうっすら動いている]   │
│                                     │
│       TANK TAKTIX                   │
│                                     │
│   ┌─────────────────────────┐      │
│   │  Player name            │      │
│   └─────────────────────────┘      │
│   [ ENTER LOBBY ]                   │
│                                     │
└─────────────────────────────────────┘
```

- 裏でデモ試合（ボット or リプレイ）がCanvasで動く
- ログインUIは半透明パネルでオーバーレイ
- イラスト素材不要・エンジンがそのまま活きる
- 「こんなゲームだ」が一瞬で伝わる動的な第一印象

**参考：** diep.io（裏で試合が流れるタイトル画面）
**非参考：** TankMatch（静的タンクイラスト＋スタンプ体タイトル）

**実装メモ**
- デモ用のマップ・ボットをバックグラウンドで動かす、または観戦モードのストリームを流す
- Canvasはフルスクリーンで描画し、UIはDOMでその上に重ねる（現構成でも対応可能）
- UIパネルは `backdrop-filter: blur` で背景をぼかすと洗練される

**デモ映像の方針（確定）**
- プレイヤーがいる試合があれば → その試合を観戦モードで流す
- 誰もいなければ → 世界（マップ）だけ静かに表示するフォールバック

## 3. アセット戦略（Canvas → 画像化ロードマップ）

### 方針：「全部Canvas」と「全部画像」の中間

| 要素 | 方針 | 理由 |
|---|---|---|
| タンク（hull / turret） | **画像化**（スプライト） | 表現力・アニメーション |
| 地形（壁・床） | **画像化**（タイル） | 見た目の密度感 |
| HUDフレーム・パネル | **画像化** | 重厚感・デザイン自由度 |
| ミニマップ | **Canvas継続** | プロシージャル生成が必要 |
| 弾・爆発エフェクト | **Canvas継続** | 動的でリアルタイム変化 |
| ボタン・アイコン | **CSS/SVG** | 解像度フリー・軽量 |

### スプライト化ステップ

```
Step 1: タンク hull / turret を PNG 化（チーム色2種 × hull + turret）
Step 2: HUDフレームを画像化（ライフバー枠・ミニマップ枠）
Step 3: 地形タイルをテクスチャ化（壁 / 草 / 水 / 家）
```

## 4. SE / 音響設計

### 設計思想

**BGMなし**。理由：配信・実況を聞きながらプレイしてもらいたいため。
SEのみで世界観と手応えを表現する。

> ミュートボタン実装は本流ロードマップへ（SE全体のON/OFF）

### SFX 一覧（案）

| イベント | SE | 優先度 | 実装状況 |
|---|---|---|---|
| 射撃・ボム | 砲撃音（短く鋭い） | 高 | ✅ |
| 爆発 | 爆発音 | 高 | ✅ |
| アイテム取得 | ポジティブな短音 | 中 | ✅ |
| フラッグ取得 | アイテム取得と同じ音 | 中 | ✅ |
| フラッグ返還 | 達成感のある音 | 中 | ✅ |
| ボタン操作 | UI 操作・ホバー音（軽め） | 低 | ✅ |

> ※ 初期構想にあった被弾音・ゲーム終了（リザルト）音は、ゲームプレイのテンポ・配信への影響を考慮して不採用としました。

### 音の方向性

**中間路線**：ゲーム的な手応えがありながら、リアルすぎない。

- 射撃・爆発は「ズシッとした感触」がある、でも戦場の恐怖感ではない
- カートゥーンのような軽さはなく、ちゃんとタンク戦ゲームの重みがある
- 参考イメージ：ミニチュア感のある戦場、World of Tanks Blitz よりはカジュアル

### 実装方針

- Web Audio API ベース（`<audio>` タグより制御しやすい）
- アセット形式：OGG + MP3（ブラウザ互換）
- 音量調整：ミュートボタン（本流ロードマップ）
